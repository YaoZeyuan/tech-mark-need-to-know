# 态马商战决策系统使用指南

## 1. 系统用途

本系统用于辅助态马商战团队完成季度经营决策。它不会在本机模拟比赛结果，也不会替代比赛系统提交决策；它负责把教材规则、季度报告、角色输入、AI 建议和最终决策书整理成一个可复用的工作台。

你可以用它完成：
- 录入每季度财务、库存、市场和竞争对手报告。
- 让 CEO、CMO、COO、CFO、CTO 分别维护本角色决策。
- 检查关键输入是否完整、是否存在明显冲突。
- 生成角色 Prompt，复制给外部 AI，或通过 OpenAI 兼容 API 调用模型。
- 汇总最终季度决策书，并导出 Markdown 或 JSON。

## 2. 推荐使用流程

1. 打开总览页，确认当前季度和数据完整度。
2. 在报告页录入上一季度的财务、库存、市场研究和竞争对手数据。
3. 各角色进入自己的角色页，填写本季度建议和备注。
4. 在角色页展开“AI 辅助决策”，生成并复制本角色 Prompt。
5. 如需集中调用模型，进入 Prompt 工作台并选择对应角色。
6. CEO 查看校验提示、角色建议和 AI 回复，填写最终决策摘要。
7. 在决策书页检查所有字段，并导出最终决策 Markdown。
8. 在设置页导出 JSON，作为跨设备备份或团队同步文件。

## 3. 如何切换到后续季度

页面右上或报告页中的季度输入会更新当前季度。系统会把当前季度同步到 URL 的 `q` 参数，例如：

```text
/reports?q=2
/readme?q=2
```

你可以直接分享带 `q` 参数的地址。其他人打开后，会优先进入该季度。

切换季度时注意：
- 季度报告按季度保存。
- 决策书字段代表当前正在维护的本轮决策。
- 导入 JSON 会覆盖当前浏览器中的业务状态。

## 4. 页面分工

- 总览：查看当前季度、角色完成状态、关键风险和校验摘要。
- CEO：汇总经营目标、角色建议、AI 建议和最终决策。
- CMO：维护价格、促销、需求、市场份额和销售代表。
- COO：维护发货、生产、库存、生产地和物流相关判断。
- CFO：维护现金、负债上限、回购、利润预测和财务风险。
- CTO：维护产品参数、研发目标、研发预算和市场理想值。
- 决策书：汇总所有可提交字段并导出 Markdown。
- 报告：录入财务报告、库存、市场研究和竞争数据。
- Prompt：集中查看角色 Prompt、手动回复、模型回复和汇总 Prompt。
- 设置：维护模型、模板、成本规则、基础配置和导入导出。
- 态马说明：阅读本地打包的使用指南和取胜说明。

## 5. 如何录入报告

每轮比赛结果出来后，先进入报告页：

- 财务报告：录入损益表、资产负债表和系统管理员通知。
- 生产库存：录入期初库存、本期产量、期末库存和毛利率。
- 市场研究：录入市场销量、订单量、市场份额和预订份额。
- 竞争数据：录入其他小组价格、份额、销售代表、促销和品牌知名度。
- 产品参数：维护有效产品参数和市场理想值。

报告数据会进入 Prompt，帮助 AI 理解上一季度结果和当前约束。

## 6. 如何生成 Prompt

每个角色页都有默认折叠的“AI 辅助决策”区域。推荐流程：

1. 先填写本角色表单。
2. 查看校验提示，处理明显错误。
3. 展开“AI 辅助决策”。
4. 点击生成角色 Prompt。
5. 复制 Prompt 到 DeepSeek、Gemini、ChatGPT 等外部 AI。
6. 将 AI 回复回填到系统，供 CEO 汇总。

如果已经配置 OpenAI 兼容 API，可以进入 Prompt 工作台集中运行三个参谋模型和一个决策模型。

## 7. 如何导入导出

设置页提供三类常用导出：

- 导出 JSON：保存当前全部业务状态，适合备份和跨设备同步。
- 导入 JSON：用文件内容覆盖当前浏览器状态。
- 导出 Markdown：导出最终季度决策书，适合提交前检查或团队讨论。

建议每个季度提交前导出一次 JSON 和 Markdown。

## 8. 模型配置

系统使用一组共享 OpenAI 兼容网关配置：

- `baseUrl`
- `endpoint`
- `API_KEY`

参谋模型1、参谋模型2、参谋模型3、决策模型只需要分别配置 `model` 和备注。

API Key 只保存在本机浏览器 LocalStorage。AI 请求结果会缓存在 IndexedDB 中，相同请求内容可以命中缓存。

## 9. 常见问题

### 刷新页面会丢数据吗？

不会。系统会定期把业务状态保存到 LocalStorage，刷新后优先恢复本地数据。

### 可以多人同时编辑吗？

首版没有后端协同。建议一个人维护主数据，其他人通过导出 JSON 或 Markdown 进行同步。

### 系统会自动算出最优决策吗？

不会。系统只做结构化录入、校验和 AI 辅助分析。最终决策仍需要团队判断。

### `/readme` 断网能看吗？

能。说明页内容来自本地打包资源，不依赖运行时访问 GitHub。
