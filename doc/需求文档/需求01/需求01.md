# 态马商战决策系统需求文档 - 需求01

状态：待确认，确认前不进入开发。

资料来源：
- 《态马商战教材2026版.pdf》，该 PDF 为图片/矢量版，无可抽取文本，已渲染唯一页后逐页阅读。
- 用户原始需求：基于 Vue3 + TypeScript + Vite + Arco Design，实现前端决策系统，为 CEO、CMO、COO、CFO、CTO 分别创建页面；收集数据后生成角色决策 prompt 和汇总 prompt；prompt 模板与模型配置存储在 LocalStorage；支持 OpenAI 兼容 API 配置。

## 1. 完整性结论

当前原始需求方向正确，但不完整。若直接开发，只能做出“角色页面 + prompt 生成器”，无法真正帮助团队按教材完成季度经营决策。

需要补充的核心内容：
- 态马商战的业务对象：国家、产品、季度、市场、生产、营销、财务、研发、竞争对手。
- 每个角色的具体决策输入项、报表视图和校验规则。
- 季度决策书的数据结构：价格、发货量、推广预算、销售代表、时间分配、研发项目、下期生产地点、负债额度、股票回购、预测利润。
- 季度报告/市场研究报告的数据录入结构，用于形成下一轮决策闭环。
- AI prompt 必须包含结构化业务数据，而不是只要求 AI 泛泛分析。
- 设置页除了模型 API 配置，还需要 prompt 模板配置、数据导入导出和本地重置能力。

## 2. 产品目标

构建一个前端本地决策辅助系统，帮助态马商战团队把教材规则、季度输入、季度报告、角色分析和 AI 辅助建议组织成可执行的下一季度决策书。

系统目标：
- 降低成员理解教材和整理数据的压力。
- 为 CEO、CMO、COO、CFO、CTO 提供各自决策所需的数据表、图表和检查清单。
- 将收集到的数据生成标准化 prompt，分别交给多个 AI 参谋分析。
- 汇总多个 AI 的回复，生成下一步操作指南、决策建议和理由。
- 所有配置、模板、业务数据在首版中存储于 LocalStorage。

## 3. 首版范围

首版必须实现：
- 角色页面：`/ceo`、`/cmo`、`/coo`、`/cfo`、`/cto`。
- 统一首页/总览页：`/`，展示当前季度、数据完整度、角色完成状态、关键风险。
- 决策书页：`/decision-book`，汇总所有角色输入，形成一份季度决策书。
- 报告录入页：`/reports`，录入季度财务报告和市场研究报告。
- Prompt 工作台：`/prompts`，生成角色决策 prompt、三模型参谋 prompt、汇总 prompt。
- 设置页：`/settings`，配置模型、prompt 模板、数据导入导出、LocalStorage 重置。

首版不要求实现完整游戏模拟器，但必须能结构化录入教材中影响决策的关键数据，并能把这些数据完整带入 prompt。

## 4. 核心业务概念

### 4.1 国家与市场

系统内置三个市场：
- 尤国，代码 `E`，总部所在地，货币为尤元。
- 纳国，代码 `L`，货币为纳镑。
- 尼国，代码 `N`，货币为尼元。

教材关键规则：
- 公司位于尤国，初始只能在尤国生产。
- 产品在尤国、纳国、尼国三个市场销售。
- 纳国和尼国涉及汇率、通货膨胀、进口关税和本地生产偏好。
- 纳国消费者集中在较富裕人群，政府鼓励“购买国货”，本地生产同类产品销量可能高于进口产品。
- 尼国消费者偏好设计出众、品牌知名度高的产品。

初始市场参考数据：
- 纳国汇率：4 纳镑 = 1 尤元。
- 尼国汇率：2 尼元 = 1 尤元。
- 预期通胀率：尤国 0%，纳国 5%，尼国 1.25%。
- 应收账款回收期：尤国 15 天，纳国 60 天，尼国 30 天。

### 4.2 产品

系统内置三种雷墨磁产品：
- 雷墨磁1：用于汽车、卡车、小型挖掘机等重型机械，也可用于洗衣机、烘干机、洗碗机等家电，偏恶劣终端环境。
- 雷墨磁2：用于高档电子消费品、音响器材、仪器仪表，需求预计增长。
- 雷墨磁3：通用电子元件，低价、无品牌、批发型产品，通常不安排广告促销。

产品参数：
- 每个产品有两个性能指标：扭矩、电阻。
- 参数范围 1.0 到 5.0，只允许一位小数。
- 初始三个产品的扭矩和电阻均为 3.0。
- 每个产品在同一季度只能有一组产品特性，无法同时满足不同国家市场的不同理想值。

教材初始市场理想值：

| 市场 | 产品 | 扭矩理想值 | 电阻理想值 |
| --- | --- | ---: | ---: |
| 尤国 | 雷墨磁1 | 4.3 | 3.8 |
| 尤国 | 雷墨磁2 | 2.4 | 2.2 |
| 尤国 | 雷墨磁3 | 1.5 | 3.5 |
| 纳国 | 雷墨磁1 | 4.2 | 4.3 |
| 纳国 | 雷墨磁2 | 2.6 | 2.5 |
| 纳国 | 雷墨磁3 | 1.0 | 1.5 |
| 尼国 | 雷墨磁1 | 3.8 | 2.1 |
| 尼国 | 雷墨磁2 | 3.4 | 2.6 |
| 尼国 | 雷墨磁3 | 4.7 | 1.4 |

系统需要提示：市场理想值会随时间变化，营销也可能影响市场对产品性能的偏好。

### 4.3 季度

教材以一个财务季度作为标准决策周期，一个季度为三个月。团队预计需要 5 到 6 个财务季度扭转局面。

系统需要支持：
- 当前季度编号。
- 多季度历史数据。
- 本季度决策输入。
- 上季度报告录入。
- 下一季度生产地等滞后生效决策。

## 5. 角色与页面

### 5.1 CEO 页面 `/ceo`

目标：统一经营判断，检查各角色输入是否互相匹配，形成最终决策。

必须展示：
- 当前季度经营总览：利润、ROA、ROE、杠杆、现金、负债、市场份额、库存风险。
- 数据完整度：CMO、COO、CFO、CTO 是否已完成输入。
- 决策冲突提示：例如 CMO 预测需求大于 COO 发货能力，CTO 研发预算超过 CFO 资金承受能力。
- 五角色建议汇总。
- 最终决策摘要和下一步行动指南。

CEO 决策引导：
- 本季度核心目标：盈利、现金安全、市场份额、产品升级、长期 ROE 中的优先级。
- 是否接受高杠杆扩张。
- 是否优先抢占某国家/某产品市场。
- 是否调整下季度生产布局。
- 是否采纳 AI 汇总建议。

### 5.2 CMO 页面 `/cmo`

目标：负责市场、价格、推广、销售队伍和竞争态势分析。

必须展示/录入：
- 产品 × 市场的价格。
- 产品 × 市场的发货/需求/订单/市场份额。
- 产品 × 市场的广告促销预算，雷墨磁3默认不可促销。
- 各市场销售代表总人数。
- 各市场销售代表在三种产品上的时间分配。
- 各市场销售代表上限。
- 竞争对手价格、市场份额、销售代表规模、品牌知名度。
- 行业需求总量和历史趋势。

初始参考数据：

| 产品 | 初始销售价格，每箱尤元 |
| --- | ---: |
| 雷墨磁1 | 40 |
| 雷墨磁2 | 26 |
| 雷墨磁3 | 32 |

| 市场 | 雷墨磁1需求 | 雷墨磁2需求 | 雷墨磁3需求 |
| --- | ---: | ---: | ---: |
| 尤国 | 275,000 | 225,000 | 500,000 |
| 纳国 | 250,000 | 50,000 | 30,000 |
| 尼国 | 200,000 | 165,000 | 30,000 |

历史促销预算参考：

| 市场 | 雷墨磁1 | 雷墨磁2 | 雷墨磁3 |
| --- | ---: | ---: | ---: |
| 尤国 | 450,000 | 50,000 | 0 |
| 纳国 | 125,000 | 25,000 | 0 |
| 尼国 | 450,000 | 50,000 | 0 |

初始销售代表：
- 尤国 12 人。
- 纳国 7 人。
- 尼国 10 人。

初始销售代表时间分配：
- 雷墨磁1：70%。
- 雷墨磁2：20%。
- 雷墨磁3：10%。

销售代表规则：
- 新聘销售代表费用为 50,000 尤元。
- 每个销售代表每财季费用为 150,000 尤元。
- 新聘销售代表需要培训，下一个财季才能正常工作。
- 解雇销售代表的财务影响相当于该销售代表一个财季的费用。
- 每个市场的销售代表数量受系统管理员给出的上限约束，未使用额度不会保留到下一期。

### 5.3 COO 页面 `/coo`

目标：负责生产计划、生产地、发货、物流、库存和成本判断。

必须展示/录入：
- 产品 × 市场的本期发货数量。
- 产品 × 市场的期初库存、期末库存、库存成本。
- 产品 × 生产地的本期产量。
- 产品 × 下期生产地。
- 生产地切换提醒：生产地变更在下一期生效。
- 独家生产权提醒：某承包商在某季度拥有某产品独家生产权时，其他国家承包商不能生产该产品。
- 首次在纳国/尼国生产某产品时的设备安装费。
- 每次更换生产工厂的开办费。
- 成本-产量阶梯表、运输保险关税附加成本、通胀调整。

教材关键规则：
- 第一季度所有产品必须在尤国生产。
- 第一季度后可选择在尤国、纳国、尼国生产。
- 如果要在第 T 季度由某承包商独家生产某产品，需要在第 T-1 季度获得许可。
- 首次在纳国或尼国生产某产品，需要每种产品支付 1,000,000 尤元设备安装费，计入固定资产。
- 每次更换生产工厂，有 100,000 尤元开办费，计入其他费用。
- 未售出货物每期产生相当于直接生产成本 10% 的库存费用。
- 产品发货后不能在仓库之间转移。
- 工厂不允许加班；如果订单需求超过供应能力，未满足订单会流向竞争对手。

直接成本附加规则：
- 尤国生产销往纳国：在原材料和人工成本上加 32%。
- 尤国生产销往尼国：加 18%。
- 纳国生产销往尤国：加 22%。
- 纳国生产销往尼国：加 23%。
- 尼国生产销往尤国：加 16%。
- 尼国生产销往纳国：加 52%。

### 5.4 CFO 页面 `/cfo`

目标：负责财务表现、融资、现金、杠杆、ROA/ROE 和股东回报。

必须展示/录入：
- 损益表。
- 资产负债表。
- 现金、应收账款、库存、固定资产、负债、股本、资本公积、留存收益。
- ROA、ROE、财务杠杆。
- 最高负债限额。
- 股票回购金额。
- 本期预测利润。
- 利息、税率、管理费用、折旧。
- 现金上下限和短期投资提示。

教材关键公式：
- 总资产回报率 ROA = 利润 / 总资产。
- 净资产收益率 ROE = 利润 / 股东权益。
- ROE = 利润 / 销售额 × 销售额 / 总资产 × 总资产 / 股东权益。
- 财务杠杆 = 总资产 / 股东权益。

初始资产负债表，单位千尤元：

| 项目 | 金额 |
| --- | ---: |
| 现金 | 5,000 |
| 短期投资 | 0 |
| 存货 | 0 |
| 固定资产，已扣除折旧 | 25,000 |
| 资产总计 | 30,000 |
| 负债 | 0 |
| 股本，每股面值 1 尤元 | 500 |
| 已缴资本 | 29,500 |
| 留存收益 | 0 |
| 负债和股东权益总计 | 30,000 |

教材财务规则：
- 短期投资年利率 8%。
- 债务利息单利年 25%，即每财季 6.25%。
- 公司所得税为应税收入的 38%。
- 每个小组成员每期管理费用为 200,000 尤元。
- 折旧为固定资产总值除以 40 个财务季度，第一季度为 625,000 尤元。
- 现金上限 10,000,000 尤元；无负债且超过上限的部分自动转短期投资。
- 现金下限 5,000,000 尤元；低于下限时由借款补足。
- 账户不能同时存在未偿还负债和短期投资。

### 5.5 CTO 页面 `/cto`

目标：负责产品研发、产品参数、市场理想值和技术路线选择。

必须展示/录入：
- 当前产品参数：扭矩、电阻。
- 产品 × 市场的理想参数。
- 历史理想值趋势。
- 产品定位图：扭矩为 X 轴、电阻为 Y 轴。
- 研发目标参数。
- 研发预算。
- 研发成功/失败后的备注录入。

研发规则：
- 目标参数范围 1.0 到 5.0，只允许一位小数。
- 研发是否成功取决于参数变化幅度和研发预算。
- 改变幅度越大，需要预算越多。
- 只要研发成功，库存产品的特性指标会自动更新到新参数。
- 如果缺少目标参数或预算，视为研发项目不完整，不能生成有效决策。

## 6. 季度决策书

系统需要生成一张完整的季度决策书，供团队提交或复制。

决策书字段：
- 价格：产品 × 市场。
- 本期发货数量：产品 × 市场。
- 最终用户市场推广预算：产品 × 市场，雷墨磁3禁用。
- 销售代表总人数：市场维度。
- 销售代表时间分配：市场 × 产品。
- 产品研发项目：产品 × 扭矩期望值、电阻期望值、预算。
- 下期生产地点：产品维度。
- 最高负债限额。
- 股票回购金额。
- 本期预测利润。

输入校验：
- 价格必须为整数，范围 0 到 99，单位为尤元。
- 发货数量范围 0 到 999,000，单位为箱；每种产品每季度最大产量为 2,997,000。
- 发货数量表示本期新生产并发往各国家的产品数量，前期库存不计入本期发货数量。
- 推广预算仅适用于雷墨磁1和雷墨磁2；雷墨磁3不可输入推广预算。
- 每个产品每个市场的推广预算范围 0 到 999,000 尤元。
- 销售代表总人数必须为非负整数，且不能超过该市场本期允许的最大销售代表数。
- 销售代表时间分配只允许一位小数，每个市场三种产品的合计必须等于 1.0。
- 研发参数范围 1.0 到 5.0，且只允许一位小数。
- 下期生产地点只允许选择尤国、纳国、尼国。

## 7. 季度报告录入

每轮结果出来后，系统需要录入季度报告，作为下一轮决策依据。

### 7.1 财务报告

需要录入或展示：
- 损益表：销售收入、直接成本、毛利、管理费用、库存费用、研发费用、销售费用、促销费用、利息费用、折旧、其他、营业利润、利息收入、税前利润、企业所得税、税后利润。
- 资产负债表：现金、应收账款、短期投资、存货、设备、累计折旧、负债、股本、资本公积、留存收益。
- 系统管理员通知：下季度销售代表限额、应收账款回收期、应收账款金额、有效产品参数。
- 下期生产地点确认。
- 生产和库存数据：期初库存、期初库存成本、本期产量、当期成本、期末库存、期末库存成本、当期毛利率。
- 市场数据：市场销量、订单量、销售市场份额、预订市场份额。

### 7.2 市场研究报告

需要录入或展示：
- 行业需求总量。
- 每组销售市场份额。
- 每组定价。
- 每组销售代表规模。
- 各组盈利状况。
- 各地通货膨胀率和汇率。
- 每组促销费用以及股权融资/股票活动。
- 竞争对手产品品牌知名度。
- 产品定位图和市场理想值。

## 8. AI 决策流程

系统支持两种模式：

### 8.1 手动复制模式

当未配置 API Key 时：
- 系统生成角色决策 prompt。
- 用户点击复制按钮，分别粘贴到：
  - https://chat.deepseek.com/
  - https://gemini.google.com/
  - https://chatgpt.com/
- 用户把三个 AI 回复粘贴回系统。
- 系统生成汇总 prompt，引导用户粘贴给 DeepSeek 汇总。
- 用户把汇总结果录入系统，CEO 页面展示最终建议。

### 8.2 API 直连模式

当配置 OpenAI 兼容 API 时：
- 设置页维护 4 个模型槽位：参谋模型1、参谋模型2、参谋模型3、决策模型。
- 每个模型槽位包含：API_KEY、baseUrl、model、模型名备注。
- 三个参谋模型接收相同决策 prompt，生成独立分析。
- 决策模型接收三份参谋回复和汇总模板，生成最终下一步操作指南。
- API Key 仅存储在用户本地浏览器 LocalStorage 中，界面需要做掩码展示，并提供清空配置能力。

## 9. Prompt 模板

模板存储在 LocalStorage，必须可编辑、重置为默认值。

模板类型：
- CEO 决策 prompt。
- CMO 决策 prompt。
- COO 决策 prompt。
- CFO 决策 prompt。
- CTO 决策 prompt。
- 全局参谋 prompt。
- 三模型汇总 prompt。
- 最终 DeepSeek 汇总 prompt。

每个 prompt 至少应包含：
- 当前季度。
- 公司目标和当前约束。
- 本角色相关输入表。
- 上季度报告摘要。
- 竞争态势摘要。
- 待决策字段。
- 要求 AI 输出：建议值、理由、风险、假设、需要 CEO 裁决的问题。

汇总 prompt 必须要求输出：
- 三个 AI 建议的一致点。
- 三个 AI 建议的冲突点。
- 推荐决策书字段。
- 每个角色下一步动作。
- 主要风险和备选方案。
- 需要人工确认的事项。

## 10. 数据与存储

首版使用 LocalStorage。

需要持久化：
- 当前季度。
- 公司基础信息。
- 国家、产品、规则表。
- 每季度决策书。
- 每季度财务报告。
- 每季度市场研究报告。
- 各角色草稿和备注。
- AI prompt 模板。
- AI 回复和汇总结果。
- API 模型配置。

设置页必须支持：
- 导出全部 LocalStorage 数据为 JSON。
- 从 JSON 导入。
- 重置 prompt 模板。
- 清空业务数据。
- 清空模型配置。

## 11. 报表和图表要求

系统视觉以报表和工作台为主，避免营销型页面。

建议图表：
- 产品 × 市场的价格表。
- 发货量和市场需求对比表。
- 推广预算表。
- 销售代表规模和时间分配表。
- 生产地/成本/库存表。
- 损益表和资产负债表。
- ROA/ROE/杠杆指标卡。
- 产品定位散点图：扭矩 X 轴、电阻 Y 轴。
- 市场需求历史趋势折线图。
- 市场份额对比表。
- 决策完整度检查表。

## 12. 验收标准

需求确认后进入开发时，首版至少满足：
- 用户能进入 CEO、CMO、COO、CFO、CTO、决策书、报告、Prompt、设置页面。
- 用户能录入一轮完整季度决策书。
- 用户能录入一轮季度财务报告和市场研究报告。
- 系统能按角色展示对应决策表和教材规则提示。
- 系统能检查关键输入约束，并提示不完整/冲突项。
- 系统能生成角色 prompt、参谋 prompt 和汇总 prompt。
- 未配置 API Key 时，系统可通过复制 prompt 和粘贴回复完成闭环。
- 配置 API Key 后，系统可向参谋模型和决策模型发起请求。
- 所有模板、配置和业务数据刷新后仍保留在 LocalStorage。
- 提供 JSON 导入导出能力，避免误清空后无法恢复。
