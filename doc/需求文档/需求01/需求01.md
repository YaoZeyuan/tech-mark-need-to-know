# 态马商战决策系统需求文档 - 需求01

状态：可开发版，等待启动实现。

资料来源：
- 《态马商战教材2026版.pdf》，该 PDF 为图片/矢量版，已渲染唯一页后逐页阅读。
- 用户原始需求与后续补充说明。

## 1. 项目背景

本项目用于辅助态马商战团队进行季度经营决策。用户作为前端工程师，不熟悉商业经营决策内容，因此系统的核心价值不是在本机模拟游戏结果，而是把教材规则、季度数据、报告数据和角色判断结构化整理后，提供给 AI 进行辅助决策。

系统需要帮助团队完成：
- 收集每个角色的决策输入。
- 维护教材中的固定规则和可变规则表。
- 录入季度财务报告、市场研究报告和竞争对手数据。
- 生成角色决策 prompt、参谋模型 prompt 和汇总 prompt。
- 支持手动复制到外部 AI，也支持 OpenAI 兼容 API 直连。
- 导出/导入当前状态，方便不同电脑之间同步。

## 2. 技术约束

- 项目基于 Vue3 + TypeScript + Vite。
- UI 组件库使用 Arco Design。
- 路由使用 `vue-router`。
- 请求缓存 hash 使用 `md5`。
- 首版为纯前端项目，不依赖后端服务。
- API 直连模式从浏览器发起请求，默认模型服务已支持 CORS。
- 业务枚举、教材规则和初始值尽量使用常量维护；关键业务常量需要中文注释说明含义和来源。
- 开发完成后需要运行构建校验，并提交有意义的 Git commit。

## 3. 首版范围

必须实现的页面：
- `/`：总览页，展示当前季度、数据完整度、角色完成状态和关键风险。
- `/ceo`：CEO 决策页。
- `/cmo`：CMO 决策页。
- `/coo`：COO 决策页。
- `/cfo`：CFO 决策页。
- `/cto`：CTO 决策页。
- `/decision-book`：季度决策书页。
- `/reports`：季度报告录入页。
- `/prompts`：Prompt 工作台。
- `/settings`：设置页，包含模型配置、模板配置、规则表维护、导入导出和重置能力。

首版非目标：
- 不实现完整经营模拟器。
- 不在本机自动计算最终利润、现金流、ROA/ROE、需求变化或比赛结果。
- 不导出图片。
- 不接入后端数据库。

首版需要做的确定性校验：
- 必填项完整性。
- 数值范围。
- 表格合计关系。
- 角色输入之间的基础冲突提示。
- 规则表是否存在、是否可注入 prompt。

## 4. 核心业务概念

### 4.1 国家与市场

系统内置三个市场：

| 市场 | 代码 | 货币 | 说明 |
| --- | --- | --- | --- |
| 尤国 | E | 尤元 | 公司总部所在地，初始生产地 |
| 纳国 | L | 纳镑 | 发展中国家，本地生产偏好明显 |
| 尼国 | N | 尼元 | 消费者偏好设计出众和品牌知名度 |

初始市场参考数据：

| 项目 | 尤国 | 纳国 | 尼国 |
| --- | ---: | ---: | ---: |
| 汇率，外币兑 1 尤元 | 1 | 4 | 2 |
| 预期通胀率 | 0% | 5% | 1.25% |
| 应收账款回收期 | 15 天 | 60 天 | 30 天 |

教材规则：
- 公司位于尤国，第一季度所有产品必须在尤国生产。
- 产品在尤国、纳国、尼国三个市场销售。
- 纳国和尼国涉及汇率、通货膨胀、进口关税和本地生产偏好。
- 纳国本地生产同类产品的销量可能高于进口产品。
- 尼国消费者倾向选择设计出众、品牌知名度高的产品。

### 4.3 季度

教材以一个财务季度作为标准决策周期，一个季度为三个月。系统需要支持多季度数据，但 JSON 导入时直接以导入状态覆盖当前状态，不需要做版本合并。

每个季度应包含：
- 决策书数据。
- 财务报告。
- 市场研究报告。
- 竞争小组数据。
- AI 分析结果。
- 最终决策摘要。

## 5. 数据与存储

首版以 LocalStorage 为主。AI 请求缓存可使用 IndexedDB 存储，避免 LocalStorage 空间不足。

需要持久化的数据：
- 当前季度。
- 公司基础信息。
- 角色人数配置，默认 CEO、CMO、COO、CFO、CTO 各 1 人，共 5 人。
- 国家、产品、教材规则表。
- 每季度决策书。
- 每季度财务报告。
- 每季度市场研究报告。
- 多小组竞争数据。
- 各角色草稿和备注。
- Prompt 模板。
- AI 回复和汇总结果。
- API 模型配置。
- AI 请求缓存。

导入导出：
- 支持导出当前全部业务状态为 JSON。
- 支持从 JSON 导入，导入后覆盖当前状态。
- 支持导出最终决策为 Markdown。
- 不需要导出图片。

## 6. 角色页面

### 6.1 CEO 页面 `/ceo`

目标：统一经营判断，检查各角色输入是否互相匹配，形成最终决策。

必须展示：
- 当前季度经营总览：利润、ROA、ROE、杠杆、现金、负债、市场份额、库存风险。
- 数据完整度：CMO、COO、CFO、CTO 是否已完成输入。
- 决策冲突提示：例如 CMO 预测需求大于 COO 发货能力，CTO 研发预算超过 CFO 资金承受能力。
- 五角色建议汇总。
- AI 汇总建议。
- 最终决策摘要和下一步行动指南。

决策引导：
- 本季度核心目标：盈利、现金安全、市场份额、产品升级、长期 ROE 中的优先级。
- 是否接受高杠杆扩张。
- 是否优先抢占某国家/某产品市场。
- 是否调整下季度生产布局。
- 是否采纳 AI 汇总建议。

### 6.2 CMO 页面 `/cmo`

目标：负责市场、价格、推广、销售队伍和竞争态势分析。

必须展示/录入：
- 产品 × 市场的价格。
- 产品 × 市场的发货、需求、订单、市场份额。
- 产品 × 市场的广告促销预算，雷墨磁3默认不可促销。
- 各市场销售代表总人数。
- 各市场销售代表在三种产品上的时间分配。
- 各市场销售代表上限。
- 竞争对手价格、市场份额、销售代表规模、品牌知名度。
- 行业需求总量和历史趋势。

初始销售价格：

| 产品 | 初始价格，每箱尤元 |
| --- | ---: |
| 雷墨磁1 | 40 |
| 雷墨磁2 | 26 |
| 雷墨磁3 | 32 |

初始市场需求：

| 市场 | 雷墨磁1需求 | 雷墨磁2需求 | 雷墨磁3需求 |
| --- | ---: | ---: | ---: |
| 尤国 | 275,000 | 225,000 | 500,000 |
| 纳国 | 250,000 | 50,000 | 30,000 |
| 尼国 | 200,000 | 165,000 | 30,000 |

历史促销预算参考：

| 市场 | 雷墨磁1 | 雷墨磁2 | 雷墨磁3 |
| --- | ---: | ---: | ---: |
| 尤国 | 450,000 | 50,000 | 0 |
| 纳国 | 125,000 | 25,000 | 0 |
| 尼国 | 450,000 | 50,000 | 0 |

初始销售代表：
- 尤国 12 人。
- 纳国 7 人。
- 尼国 10 人。

初始销售代表时间分配：
- 雷墨磁1：70%。
- 雷墨磁2：20%。
- 雷墨磁3：10%。

销售代表规则：
- 新聘销售代表费用为 50,000 尤元。
- 每个销售代表每财季费用为 150,000 尤元。
- 新聘销售代表需要培训，下一个财季才能正常工作。
- 解雇销售代表的财务影响相当于该销售代表一个财季的费用。
- 每个市场的销售代表数量受系统管理员给出的上限约束，未使用额度不会保留到下一期。

### 6.3 COO 页面 `/coo`

目标：负责生产计划、生产地、发货、物流、库存和成本判断。

必须展示/录入：
- 产品 × 市场的本期发货数量。
- 产品 × 市场的期初库存、期末库存、库存成本。
- 产品 × 生产地的本期产量。
- 产品 × 下期生产地。
- 生产地切换提醒：生产地变更在下一期生效。
- 独家生产权提醒。
- 首次在纳国/尼国生产某产品时的设备安装费。
- 每次更换生产工厂的开办费。
- 成本-产量阶梯表、运输保险关税附加成本、通胀调整。

生产规则：
- 第一季度所有产品必须在尤国生产。
- 第一季度后可选择在尤国、纳国、尼国生产。
- 如果要在第 T 季度由某承包商独家生产某产品，需要在第 T-1 季度获得许可。
- 首次在纳国或尼国生产某产品，需要每种产品支付 1,000,000 尤元设备安装费，计入固定资产。
- 每次更换生产工厂，有 100,000 尤元开办费，计入其他费用。
- 未售出货物每期产生相当于直接生产成本 10% 的库存费用。
- 产品发货后不能在仓库之间转移。
- 工厂不允许加班；如果订单需求超过供应能力，未满足订单会流向竞争对手。

直接成本附加规则：
- 尤国生产销往纳国：在原材料和人工成本上加 32%。
- 尤国生产销往尼国：加 18%。
- 纳国生产销往尤国：加 22%。
- 纳国生产销往尼国：加 23%。
- 尼国生产销往尤国：加 16%。
- 尼国生产销往纳国：加 52%。

成本-产量阶梯表要求：
- 首版内置教材中的全部成本-产量阶梯表。
- 规则表只用于展示、维护和注入 prompt，不在本机自动测算利润。
- 设置页需要允许用户维护成本-产量表。

### 6.4 CFO 页面 `/cfo`

目标：负责财务表现、融资、现金、杠杆、ROA/ROE 和股东回报。

必须展示/录入：
- 损益表。
- 资产负债表。
- 现金、应收账款、库存、固定资产、负债、股本、资本公积、留存收益。
- ROA、ROE、财务杠杆。
- 最高负债限额。
- 股票回购金额。
- 本期预测利润。
- 利息、税率、管理费用、折旧。
- 现金上下限和短期投资提示。

财务公式：
- 总资产回报率 ROA = 利润 / 总资产。
- 净资产收益率 ROE = 利润 / 股东权益。
- ROE = 利润 / 销售额 × 销售额 / 总资产 × 总资产 / 股东权益。
- 财务杠杆 = 总资产 / 股东权益。

初始资产负债表，单位千尤元：

| 项目 | 金额 |
| --- | ---: |
| 现金 | 5,000 |
| 短期投资 | 0 |
| 存货 | 0 |
| 固定资产，已扣除折旧 | 25,000 |
| 资产总计 | 30,000 |
| 负债 | 0 |
| 股本，每股面值 1 尤元 | 500 |
| 已缴资本 | 29,500 |
| 留存收益 | 0 |
| 负债和股东权益总计 | 30,000 |

财务规则：
- 短期投资年利率 8%。
- 债务利息单利年 25%，即每财季 6.25%。
- 公司所得税为应税收入的 38%。
- 每个小组成员每期管理费用为 200,000 尤元。
- 默认角色人数为 5 人，用户可在设置页维护具体角色人数。
- 折旧为固定资产总值除以 40 个财务季度，第一季度为 625,000 尤元。
- 现金上限 10,000,000 尤元；无负债且超过上限的部分自动转短期投资。
- 现金下限 5,000,000 尤元；低于下限时由借款补足。
- 账户不能同时存在未偿还负债和短期投资。

### 6.5 CTO 页面 `/cto`

目标：负责产品研发、产品参数、市场理想值和技术路线选择。

必须展示/录入：
- 当前产品参数：扭矩、电阻。
- 产品 × 市场的理想参数。
- 历史理想值趋势。
- 产品定位图：扭矩为 X 轴、电阻为 Y 轴。
- 研发目标参数。
- 研发预算。
- 研发成功/失败后的备注录入。

研发规则：
- 目标参数范围 1.0 到 5.0，只允许一位小数。
- 研发是否成功取决于参数变化幅度和研发预算。
- 改变幅度越大，需要预算越多。
- 只要研发成功，库存产品的特性指标会自动更新到新参数。
- 如果缺少目标参数或预算，视为研发项目不完整，不能生成有效决策。

## 7. 季度决策书 `/decision-book`

系统需要生成完整季度决策书，供团队提交、复制或导出。

决策书字段：
- 价格：产品 × 市场。
- 本期发货数量：产品 × 市场。
- 最终用户市场推广预算：产品 × 市场，雷墨磁3禁用。
- 销售代表总人数：市场维度。
- 销售代表时间分配：市场 × 产品。
- 产品研发项目：产品 × 扭矩期望值、电阻期望值、预算。
- 下期生产地点：产品维度。
- 最高负债限额。
- 股票回购金额。
- 本期预测利润。

输入校验：
- 价格必须为整数，范围 0 到 99，单位为尤元。
- 发货数量范围 0 到 999,000，单位为箱。
- 每种产品每季度最大产量为 2,997,000。
- 发货数量表示本期新生产并发往各国家的产品数量，前期库存不计入本期发货数量。
- 推广预算仅适用于雷墨磁1和雷墨磁2；雷墨磁3不可输入推广预算。
- 每个产品每个市场的推广预算范围 0 到 999,000 尤元。
- 销售代表总人数必须为非负整数，且不能超过该市场本期允许的最大销售代表数。
- 销售代表时间分配只允许一位小数，每个市场三种产品的合计必须等于 1.0。
- 研发参数范围 1.0 到 5.0，且只允许一位小数。
- 下期生产地点只允许选择尤国、纳国、尼国。

## 8. 季度报告录入 `/reports`

每轮结果出来后，系统需要录入季度报告，作为下一轮决策依据。

### 8.1 财务报告

需要录入或展示：
- 损益表：销售收入、直接成本、毛利、管理费用、库存费用、研发费用、销售费用、促销费用、利息费用、折旧、其他、营业利润、利息收入、税前利润、企业所得税、税后利润。
- 资产负债表：现金、应收账款、短期投资、存货、设备、累计折旧、负债、股本、资本公积、留存收益。
- 系统管理员通知：下季度销售代表限额、应收账款回收期、应收账款金额、有效产品参数。
- 下期生产地点确认。
- 生产和库存数据：期初库存、期初库存成本、本期产量、当期成本、期末库存、期末库存成本、当期毛利率。
- 市场数据：市场销量、订单量、销售市场份额、预订市场份额。

### 8.2 市场研究报告

教材会提供其他小组/竞争对手相关数据时，系统需要支持录入；没有提供的数据可留空，并在 prompt 中标记为未知。

需要录入或展示：
- 行业需求总量。
- 每组销售市场份额。
- 每组定价。
- 每组销售代表规模。
- 各组盈利状况。
- 各地通货膨胀率和汇率。
- 每组促销费用以及股权融资、股票活动。
- 竞争对手产品品牌知名度。
- 产品定位图和市场理想值。

## 9. AI 决策流程 `/prompts`

### 9.1 手动复制模式

当未配置 API Key 时：
- 系统生成角色决策 prompt。
- 用户点击复制按钮，分别粘贴到 DeepSeek、Gemini、ChatGPT。
- 用户把三个 AI 回复粘贴回系统。
- 系统生成汇总 prompt，引导用户粘贴给 DeepSeek 汇总。
- 用户把汇总结果录入系统，CEO 页面展示最终建议。

外部 AI 地址：
- https://chat.deepseek.com/
- https://gemini.google.com/
- https://chatgpt.com/

### 9.2 API 直连模式

当配置 OpenAI 兼容 API 时：
- 设置页维护 4 个模型槽位：参谋模型1、参谋模型2、参谋模型3、决策模型。
- 每个模型槽位包含：API_KEY、baseUrl、endpoint、model、模型名备注。
- 默认 endpoint 为 `/v1/chat/completions`，允许用户配置 endpoint，配置后覆盖默认值。
- API Key 仅存储在本机浏览器 LocalStorage 中，界面需要掩码展示，并提供清空配置能力。
- 参谋模型1、参谋模型2、参谋模型3并发接收相同决策 prompt，生成独立分析。
- 三个参谋模型全部返回后，决策模型接收三份参谋回复和汇总模板，生成最终下一步操作指南。
- 请求内容使用 md5 做缓存 key；命中缓存时直接返回缓存结果。
- AI 缓存可优先使用 IndexedDB，若实现成本过高，可先使用 LocalStorage，但需要保留升级空间。

### 9.3 Prompt 模板

模板存储在 LocalStorage，必须可编辑、可重置为默认值。

模板类型：
- CEO 决策 prompt。
- CMO 决策 prompt。
- COO 决策 prompt。
- CFO 决策 prompt。
- CTO 决策 prompt。
- 全局参谋 prompt。
- 三模型汇总 prompt。
- 最终 DeepSeek 汇总 prompt。

每个 prompt 至少应包含：
- 当前季度。
- 公司目标和当前约束。
- 本角色相关输入表。
- 上季度报告摘要。
- 竞争态势摘要。
- 教材规则和成本-产量表摘要。
- 待决策字段。
- 要求 AI 输出：建议值、理由、风险、假设、需要 CEO 裁决的问题。

汇总 prompt 必须要求输出：
- 三个 AI 建议的一致点。
- 三个 AI 建议的冲突点。
- 推荐决策书字段。
- 每个角色下一步动作。
- 主要风险和备选方案。
- 需要人工确认的事项。

## 10. 设置页 `/settings`

设置页必须支持：
- 模型槽位配置：参谋模型1、参谋模型2、参谋模型3、决策模型。
- Prompt 模板编辑和恢复默认。
- 成本-产量表维护。
- 市场、产品、角色人数等基础配置维护。
- 导出全部状态为 JSON。
- 从 JSON 导入并覆盖当前状态。
- 导出最终决策为 Markdown。
- 清空业务数据。
- 清空模型配置。
- 清空 AI 缓存。

## 11. 报表和图表要求

系统视觉以报表和工作台为主，避免营销型页面。

建议图表：
- 产品 × 市场的价格表。
- 发货量和市场需求对比表。
- 推广预算表。
- 销售代表规模和时间分配表。
- 生产地、成本、库存表。
- 损益表和资产负债表。
- ROA、ROE、杠杆指标卡。
- 产品定位散点图：扭矩 X 轴、电阻 Y 轴。
- 市场需求历史趋势折线图。
- 市场份额对比表。
- 决策完整度检查表。

## 13. 验收标准

首版至少满足：
- 可以进入所有指定页面。
- 可以录入一轮完整季度决策书。
- 可以录入一轮季度财务报告和市场研究报告。
- 可以维护角色人数、模型配置、prompt 模板和成本-产量表。
- 可以录入多小组竞争数据。
- 可以按角色展示对应决策表和教材规则提示。
- 可以检查关键输入约束，并提示不完整或冲突项。
- 可以生成角色 prompt、参谋 prompt 和汇总 prompt。
- 未配置 API Key 时，可以通过复制 prompt 和粘贴回复完成闭环。
- 配置 API Key 后，可以并发调用三个参谋模型，再调用决策模型汇总。
- 相同 AI 请求内容可以命中 md5 缓存。
- 所有模板、配置和业务数据刷新后仍保留。
- 可以导出和导入 JSON。
- 可以导出最终决策 Markdown。
- 构建命令通过。
