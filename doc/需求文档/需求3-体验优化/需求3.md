# 需求3 - 体验优化

状态：可开发版。

## 1. 说明页 `/readme`

系统需要增加说明页，路径为 `/readme`，用于向团队成员解释态马商战决策系统的使用方式。

目标：
- 说明页需要能从应用导航进入。
- `/readme?q=2` 等带季度 query 的地址需要正常打开，不应跳回首页或空白页。
- 内容使用本地打包方式，不在运行时依赖 GitHub raw Markdown。
- 内容应基于 `doc/项目说明/使用指南.md`，可以重写为更适合阅读的 Markdown，也可以拆分为必要章节。
- `使用指南` 应为本项目的使用指南。
- 应增加文档，基于现有使用指南产出对态马商战的快速简介和取胜说明。
- 内容至少需要覆盖：系统用途、页面分工、如何切换后续季度、如何录入报告、如何生成 Prompt、如何导入导出数据。
- 导航入口、router 路由和构建产物需要保持一致。

## 2. CTO 产品定位图联动

CTO 页面中，产品定位图需要跟随研发项目中的目标参数变化。

要求：
- 产品定位图同时展示当前产品参数和研发目标参数。
- 修改研发项目中的目标扭矩、目标电阻后，图表需要立即响应。
- 当前参数点和目标参数点需要用颜色或图例区分。
- 可以使用连线表示从当前参数到目标参数的研发方向。
- 图表仍需要展示产品名称，避免用户无法对应产品。

验收：
- 修改任一产品的目标扭矩或目标电阻，图表中的目标点同步移动。
- 当前参数点不应被误改为研发目标点。
- 图例或视觉样式能明确区分“当前”和“目标”。

## 3. 角色页内 AI 辅助决策

生成角色 Prompt 的入口应放到对应角色页面中，让用户填完输入后能直接知道下一步动作。

要求：
- CEO、CMO、COO、CFO、CTO 页面都需要增加默认折叠的“AI 辅助决策”区域。
- 折叠区域内需要能生成当前角色的角色 Prompt。
- 折叠区域内需要支持复制 Prompt。
- 使用 Arco Steps 步骤条引导用户完成：填写输入、查看校验、生成角色 Prompt、复制或调用 AI、回填建议。
- Steps 只做操作引导，不需要强制阻断用户操作。
- 角色页生成的 Prompt 与 `/prompts` 页面生成的同角色 Prompt 需要保持一致。
- `/prompts` 页面继续保留为集中工作台，角色页负责就地引导，`/prompts` 负责集中查看和批量操作。

## 4. 其他易用性检查范围

本需求中的“检查还有哪些不易操作或使用的地方，一并修改”限定为本节范围，避免变成无边界重构。

本次只处理明确可验收的体验问题：
- `/readme` 路由或导航不可用。
- 表单填写完成后缺少下一步引导。
- Prompt 入口离角色输入页太远。
- CTO 产品定位图不响应研发目标参数。
- GitHub Fork Ribbon 作为说明入口时遮挡主要操作。

不纳入本次：
- 大规模视觉重做。
- 全部页面信息架构重排。
- 新增经营模拟器。
- 新增后端或登录权限。
- 与本需求无直接关系的性能优化。

## 5. GitHub Fork Ribbon 保留

模板代码需要保留。

要求：
- 不应移除 `index.html` 中的 GitHub Fork Ribbon 样式和链接。
- 页面首屏应出现 `Fork me on GitHub`，作为介绍入口。
- Ribbon 需要保持可点击。
- Fork Ribbon 点击后继续跳转 GitHub 仓库首页。
- Ribbon 不应遮挡核心导航、角色表单、弹窗按钮或移动端顶部内容。

## 6. 项目 README

除了重写 `使用指南`，也应为项目编写 `README.md`，方便大家快速了解项目情况和使用方式。

要求：
- `README.md` 面向项目协作者，说明项目用途、启动方式、构建方式和主要页面。
- `README.md` 需要说明 `pnpm build` 会额外复制 `readme.html` 到 `dist`。
- `README.md` 需要说明首版数据存储在 LocalStorage，AI 缓存在 IndexedDB。

## 7. 验收标准

- `/readme?q=2` 可访问，并能通过导航进入。
- `/readme` 内容来自本地打包资源，断网时仍可阅读已打包内容。
- CTO 研发目标变化时，产品定位图立即变化。
- 各角色页都有默认折叠的 AI 辅助决策区域。
- 各角色页可以生成并复制本角色 Prompt。
- `/prompts` 页面仍可用。
- Fork Ribbon 保留、点击跳转 GitHub 仓库首页，且不遮挡主要操作。
- `README.md` 存在，并说明项目用途、启动方式、构建方式和主要页面。
- 构建命令通过。
