# 需求2 - 体验优化

状态：可开发版，纳入当前实现。

## 1. 本地自动保存

- 页面输入内容需要持久化到 LocalStorage。
- 系统每 10 秒检查一次变更并写入 LocalStorage。
- 浏览器刷新或关闭前需要兜底保存未落盘变更。
- 页面重新打开时优先从 LocalStorage 恢复业务数据、模型配置和模板配置。

## 2. URL 分享参数

- 页面状态中会影响分享结果的参数需要同步到 query。
- 当前季度同步为 `q` 参数，例如 `/decision-book?q=2`。
- Prompt 工作台当前角色同步为 `role` 参数，例如 `/prompts?q=2&role=cfo`。
- 通过带 query 的 URL 打开页面时，query 参数优先于 LocalStorage 中的对应参数。
- 无效 query 参数需要忽略，不应破坏本地已有状态。

## 3. AI 模型配置

- 所有 OpenAI 兼容模型共用同一组 `baseUrl`、`endpoint` 和 `API_KEY`。
- 参谋模型1、参谋模型2、参谋模型3、决策模型只需要分别配置 `model` 和备注。
- 旧版本中每个模型槽位单独保存的 `baseUrl`、`endpoint` 和 `API_KEY`，读取时需要尽量迁移到共享配置。
- API Key 仍只保存在本机 LocalStorage，并使用密码输入框展示。

## 4. 模板代码清理

- 移除 `index.html` 中默认模板遗留的 GitHub Fork Ribbon 样式和链接。
- 页面首屏不应再出现 `Fork me on GitHub`。

## 5. 验收标准

- 编辑任意表单后刷新页面，输入内容仍保留。
- 打开 `/prompts?q=2&role=cfo` 后，页面进入第 2 季度并选中 CFO。
- 设置页只需要填写一次共享 API 网关配置，每个模型槽位只展示 model 和备注。
- API 并发分析使用共享网关配置请求四个模型槽位，并继续支持 md5 缓存。
- 构建命令通过，浏览器控制台无应用错误。
